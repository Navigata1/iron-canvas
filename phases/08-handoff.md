# Phase 8: HANDOFF
## *Optional Phase — Triggered for Client Deliverables*

> **TRIGGER CONDITIONS:**
> - Project type B, D, E, F (any client-facing site)
> - ORIENT output includes: client_handoff: true
> - Any time the build is for someone other than Jon/Island Development Crew internal use
>
> **SKIP when:**
> - Internal Iron Canvas Studio site
> - Personal portfolio or test build
> - Prototype that will not be maintained by a non-developer

---

## PURPOSE

Phase 8 generates a HANDOFF.md in the project root.
This document is written FOR the client — non-technical language.
It answers three questions:
1. What was built and why?
2. How do I update it without breaking it?
3. Who do I call when something is wrong?

It also generates an internal CHANGELOG entry, a component annotation map,
and (if applicable) a CMS recommendation.

---

## STEP 2: CMS RECOMMENDATION

After assessing the project, recommend a CMS tier if applicable:

```
NO CMS NEEDED:
  → Site content changes less than 4x per year
  → Only developer makes updates
  → Static content only (no blog, no products, no events)
  → Recommendation: Flat file editing. Document which files to edit.

LIGHTWEIGHT CMS:
  → Content changes 1-4x per month
  → Non-technical client needs to update text/images
  → Recommendation: Netlify CMS (free) or Decap CMS (headless, git-based)
  → No database needed. Edits via a UI, committed to git.

FULL HEADLESS CMS:
  → Blog, news, products, or events updated regularly
  → Multiple content editors
  → Recommendation: Sanity (flexible, best DX) or Contentful (enterprise)
  → Requires API integration — not a Phase 8 task, flag for separate engagement

E-COMMERCE:
  → Product catalog that changes regularly
  → Recommendation: Shopify headless API + Vercel frontend
  → OR: Stay fully in Shopify (simpler for non-developers)
```
