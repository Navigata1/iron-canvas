# Phase 3.9: PACKAGE — Design PRD Packaging

> **Origin:** Inspired by the Antigravity video workflow (Prompt 2 pattern).
> The Orchestrator converts all research into one unified Design PRD — the single
> document consumed by all parallel Build Agents simultaneously.
>
> A weak PRD = conflicting agent outputs that don't integrate.
> A precise PRD = coherent, unified product that passes verification first try.
>
> This is the most important document in the entire pipeline.

## The Rule

The Design PRD is the Orchestrator's primary output before agent dispatch. It contains everything a Build Agent needs to execute their domain independently — no guessing, no assumptions, no conflicts with other agents.

---

## How to Produce the Design PRD

The Orchestrator reads all Phase 1-3.5 outputs and synthesizes them into the PRD template below. Every section maps directly to a Build Agent's domain.

---

## Orchestrator Dispatch Prompt

After writing the Design PRD, the Orchestrator sends this to all Build Agents:

```markdown
Iron Canvas Build Phase: Design PRD has been packaged and is ready.

Full Design PRD is at: design-prd.md
North Star reference is at: north-star-reference.png
Technical validation report is at: technical-validation-report.md

[AGENT-A]: Execute Section 3 (token system) + Section 9 (your assignment).
           Your outputs: tokens.css, index.html, base.css
           Integration contract: Section 10

[AGENT-B]: Execute Section 4 (motion architecture) + Section 9 (your assignment).
           Reference Section 3 for all token variable names.
           Your outputs: scroll.js, load-sequence.js, [scroll-engine.js], [transitions.js]

[AGENT-C]: Execute Section 5 (component system) + Section 9 (your assignment).
           Reference Section 3 for all token variable names.
           Reference Agent-A's HTML for element selectors and data attributes.
           Your outputs: components.css, interactions.js, typography.js

[AGENT-D]: Execute Section 6 (artifact requirements) + Section 9 (your assignment).
           Complete the Pre-Generation Checklist before each prompt.
           Your outputs: artifact-prompts.md, generated images, frames/ directory

[AGENT-E]: Review ALL outputs from Agents A-D.
           Execute Section 7 (performance + accessibility).
           Your outputs: accessibility-audit.md, performance-checklist.md, qa-fixes.md

All agents: Honor Section 11 (Anti-Pattern Veto List).
All agents: Reference north-star-reference.png as the visual target.
Merge sequence after completion: A → B → C → D → E
```
