# 🎨 Iron Canvas

**Adaptive Site-Aware Design Enhancement Methodology**

> "Enhancement means amplifying what's already good, not replacing it with a uniform style."

Iron Canvas is NOT a template. It's a methodology for making any website the best version of ITSELF. Every site has its own DNA — palette, fonts, imagery style, personality, mood. Iron Canvas reads that DNA first, then enhances within it.

Three sites enhanced by Iron Canvas should look like three DIFFERENT premium sites — not three copies of the same dark template.

## Quick Start

1. Read `Iron_Canvas_v1.md` for the complete methodology
2. Use `phases/01-study.md` to begin profiling any site
3. Work through phases 2-7 sequentially
4. Reference `references/` for technical details on specific subsystems

## The 7-Phase Pipeline

```
STUDY → FEEL → COLLECT → FORGE → GENERATE → COMPOSE → REFINE
  1        2       3        4         5          6        7
```

| Phase | Name | Purpose |
|-------|------|---------|
| 1 | **STUDY** | Extract site DNA — palette, fonts, imagery, personality |
| 2 | **FEEL** | Discover what the site is trying to be emotionally |
| 3 | **COLLECT** | Find references that match THIS site's feel |
| 4 | **FORGE** | Structural upgrades that respect the existing design |
| 5 | **GENERATE** | Create context-aware artifacts (images, frames, sequences) |
| 6 | **COMPOSE** | Integrate artifacts + build scroll engine if needed |
| 7 | **REFINE** | Iterative polish until premium quality is achieved |
