# Iron_Canvas_v3 — Adaptive Site-Aware Design Enhancement
## *The Multi-Agent Design Orchestration System*

Iron Canvas is an adaptive, site-aware, multi-agent design enhancement and creation methodology. It is not a template — it is an orchestration system.

Three lineages fused into one definitive methodology:
(A) Island Development Crew production pipeline — 7-phase battle-tested system from real projects.
(B) Rigorous animation architecture — Brand Personality Matrix, GSAP motion taxonomy, Awwwards 5-axis verification.
(C) Antigravity multi-agent orchestration pattern — parallel Build Agents, Cross-LLM Technical Validation, and RPIT (Research → Plan → Implement → Test) loops.

## THE PIPELINE

### SOLO MODE (1 agent, sequential)
`ORIENT(0) → STUDY(1) → FEEL(2) → SCOUT(3) → VALIDATE(3.5) → PACKAGE(3.9) → FORGE(4) → GENERATE(5) → COMPOSE(6) → VERIFY(7)`

### MISSION MODE (Full team, maximum power)
1. **ORCHESTRATOR** (Opus): Mission Controller. Runs Phases 0-3.9. Never writes code.
2. **AGENT-A** (Foundation): Tokens, HTML scaffold, semantic structure.
3. **AGENT-B** (Motion): GSAP, scroll engine, page transitions.
4. **AGENT-C** (UI/UX): Components, 3-state interactivity, cursor, typography.
5. **AGENT-D** (Artifacts): AI image generation, context-aware prompting, scroll sequences.
6. **AGENT-E** (QA): A11y, performance, cross-browser, and 5-axis design audit.

## CORE PRINCIPLES

1. **The Golden Rule:** STUDY what's there before changing anything. Extract DNA. Understand.
2. **RPIT Loop:** For every major feature: Research domain → Plan architecture → Implement → Test.
3. **Spec-Driven Development:** The Design PRD (Phase 3.9) is the immutable source of truth for all build agents.
4. **Agentic Stack:**
   - **Claude Code:** Primary for precision engineering (`--print --permission-mode bypassPermissions`).
   - **Codex:** Primary for bulk building and refactoring (`pty:true --full-auto`).
   - **Cursor/Sub-agents:** Use `sessions_spawn` for parallel MISSION tasks.
5. **Multi-Agent Coordination:** Use git worktrees to develop Foundation, Motion, and UI in parallel without conflicts.

## PHASES OVERVIEW

- **PHASE 0: ORIENT** — Project classification and mode selection.
- **PHASE 1: STUDY** — Read the site's DNA (palette, fonts, imagery, layout).
- **PHASE 2: FEEL** — Discover the emotional target and Brand Personality Matrix.
- **PHASE 3: SCOUT** — Gather industry-matched references and generate the North Star image.
- **PHASE 3.5: VALIDATION** — Cross-LLM technical feasibility check (Antigravity pattern).
- **PHASE 3.9: PACKAGE** — Compile the Design PRD for build agent dispatch.
- **PHASE 4: FORGE** — Structural enhancement (Agent-A).
- **PHASE 5: GENERATE** — Context-aware artifact creation (Agent-D).
- **PHASE 6: COMPOSE** — Integration + GSAP ScrollTrigger/Canvas engine (Agent-B + Agent-C).
- **PHASE 7: VERIFY** — 5-axis Awwwards-grade audit and iterative refinement.
